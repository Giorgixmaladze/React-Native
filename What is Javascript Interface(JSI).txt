JSI(Javascript Interface) არის React Native ს გული და სული. თუ აქამდე React Nativeს აკრიტიკებდნენ მუშაობის გამო, JSI არის ზუსტად, რომელმაც ეს პრობლემა მოაგვარა.



1. ძველი პრობლემა: "ხიდი" (The Bridge)

    იმისათვის,რომ გავიგოთ JSI, ჯერ უნდა გავიხსენოთ,როგორ მუშაობდა React Native მანამდე. წარმოიდგინე ორი კუნძული: ერთზე ცხოვრიბს Javascript, მეორეზე კი Native კოდი(Java/Swift).მათ შორის გადებული იყო ერთი ვიწრო ხიდი.

        - როდესაც JavaScript-ს უნდოდა Native-ისთვის რამე ეთქვა, ის ინფორმაციას ათავსებდა JSON ფორმატში (ტექსტად).


        -ეს ტექსტი გადადიოდა ხიდზე.


        -მეორე მხარეს ნატივურ კოდს ეს ტექსტი უნდა ამოეკითხა, დაემუშავებინა და პასუხი ისევ JSON-ად გამოეგზავნა.

    ეს პროცესი იყო ასინქრონული და ნელი. თუ ბევრი ინფორმაცია ერთდროულად იყრებოდა ხიდზე (მაგალითად, სწრაფი სკროლვისას), ხიდი "იჭედებოდა" და აპლიკაცია კადრებს კარგავდა (Lag).

2.რა არის JSI და როგორ მუშაობს ის?

    JSI არის C++ ზე დაწერილი ფენა,რომელიც საშუალებას აძლევს JavaScript ის ძრავას (Hermes ან v8),რომ პირდაპირ დაინახოს და გამოიძახოს ნატივური მეთოდები


    აქ უკვე აღარ გვჭირდება ინფორმაციის JSON-ად ქცევა და ხიდზე გადაგზავნა. JSI-ის მეშვეობით JavaScript-ს აქვს პირდაპირი წვდომა (Reference) ნატივურ ობიექტებზე.


    *მთავარი მახასიათებლები

        -პირდაპირი კავშირი: JavaScript-მა "იცის", რომ არსებობს ნატივური ფუნქცია და შეუძლია მას პირდაპირ დააჭიროს "ღილაკზე".

        -სინქრონულობა: თუ ადრე JavaScript-ს უნდა ელოდა, როდის მოვიდოდა პასუხი ხიდიდან, ახლა მას შეუძლია ფუნქცია სინქრონულად გამოიძახოს და პასუხი მომენტალურად მიიღოს.

        -C++-ის ძალა: JSI დაწერილია C++-ზე, რაც იმას ნიშნავს, რომ ის ძალიან ახლოსაა აპარატურასთან და მუშაობს წარმოუდგენლად სწრაფად.
