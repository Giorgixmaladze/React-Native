ზოგადად, როდესაც აპლიკაციის აწყობას სწავლობენ, ბევრი inline გასტილვას იყენებს რაც მაგ ეტაპზე მისაღებია ხოლო მომავალში არა.

ჩვენ ვიცით ვებში როგორ გავსტილოთ ესა თუ ის ელემენტი, მაგრამ React Nativeში ცოტა სხვანაირადაა. ეგ უკვე განხილული გვაქვს და ჯერ ისევ inline გასტილვაზე ვართ.


მაგრამ,როდესაც მეტი სტილის წესი ემატება, კოდის წაკითხვა უფრო რთულდება. ამ პრობლემის მოსაგვარებლად, React Native-მა შექმნა StyleSheet API. StyleSheet API-ის დახმარებით, ჩვენ შეგვიძლია სტილიზაციის წესები ცალკე დავწეროთ და მათზე მივუთითოთ კომპონენტების რენდერის დროს. ისინი CSS Stylesheet-ების მსგავსად ფუნქციონირებენ და საშუალებას გვაძლევენ სტილის წესები მრავალი კომპონენტისთვის განმეორებით გამოვიყენოთ.



StyleSheet API-ის საშუალებით, ჩვენ შეგვიძლია ამოვიღოთ ხაზოვანი სტილიზაცია ჩვენი <AwesomeBox> კომპონენტიდან. ამის გაკეთება საშუალებას გვაძლევს განმეორებით გამოვიყენოთ სტილის წესები და შევინარჩუნოთ ჩვენი <AwesomeBox> კომპონენტის მარტივად წაკითხვადობა.


// StyleSheet API-ის გამოყენება
const AwesomeBox = () => (
 <View style={styles.box} />
);

const styles = StyleSheet.create({
 box: {
 width: 100,
 height: 100,
 backgroundColor: 'red'
 },
});



StyleSheet API-ის ყველაზე მნიშვნელოვანი მეთოდია create() მეთოდი, რომელიც ქმნის StyleSheet ობიექტს, რომელიც შეგვიძლია გადავცეთ ჩვენს კომპონენტებს სტილის წესების შესანახად. ზემოთ მოცემულ კოდში ჩვენ გამოვიყენეთ box, როგორც მითითება (reference) ჩვენი სტილიზაციის წესებზე.







React Native-ში (რომელიც არის ღია საწყისი კოდის მქონე ფრეიმვორკი მობილური აპლიკაციების შესაქმნელად და მუშაობს სხვადასხვა ოპერაციულ სისტემაზე, როგორიცაა iOS და Android), ჩვენ შეგვიძლია ერთ კომპონენტს რამდენიმე სტილი მივანიჭოთ style prop-ისთვის სტილის ობიექტების მასივის (array), StyleSheet რეფერენსების ან ორივეს კომბინაციის გადაცემით. ეს მიდგომა ჰგავს HTML ელემენტზე რამდენიმე CSS კლასის მინიჭებას, სადაც თითოეულ კლასს თავისი სტილების ნაკრები შემოაქვს.




მაგალითად, მხოლოდ ერთი სტილის ობიექტის მინიჭების ნაცვლად, ჩვენ შეგვიძლია გადავცეთ მასივი:
<View style={[styles.box, styles.red]} />



ან შეგვიძლია გავაერთიანოთ StyleSheet რეფერენსები (inline) სტილის ობიექტებთან:


<View style={[styles.box, { borderRadius: 10 }]} />




როდესაც სტილების მასივი გადაეცემა, React Native მათ მარცხნიდან მარჯვნივ კითხულობს და იყენებს. ამ დროს ყველაზე მარჯვენა სტილებს ენიჭებათ პრიორიტეტი და გადააწერენ (override) მასივში უფრო მარცხნივ/ადრე მყოფ ნებისმიერ კონფლიქტურ თვისებას. ამ მაგალითში, borderRadius: 10 გადააწერს ნებისმიერ კონფლიქტურ სტილს styles.box-იდან.



სტილების შერწყმა სასარგებლოა მაშინ, როდესაც გვინდა, რომ კომპონენტმა მემკვიდრეობით მიიღოს საბაზისო სტილი, მაგრამ გვჭირდება მცირედი კორექტირებები მის განსასხვავებლად. მაგალითად, შეიძლება გვქონდეს ღილაკის (button) ზოგადი სტილი, მაგრამ კონკრეტულ ღილაკზე გვინდოდეს უნიკალური ფერის ან ჩარჩოს გამოყენება. სტილების შერწყმით ჩვენ თავს ვარიდებთ კოდის დუბლირებას და ვინარჩუნებთ ჩვენი სტილიზაციის მოდულურობას.
