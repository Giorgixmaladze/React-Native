მობილური აპლიკაციები ეშვება სხვადასხვა ზომისა და ფორმის ეკრანებზე; ეკრანმა შეიძლება ფორმაც კი შეიცვალოს მოწყობილობის გადაბრუნებისას (როტაციისას). თუ ჩვენს აპლიკაციას ამ ცვლილებების მიმართ რესპონსიულს (responsive) გავხდით, გარანტირებული გვექნება, რომ ის შესანიშნავად გამოიყურება და სწორად იმუშავებს ნებისმიერ მოწყობილობაზე.



განლაგების (layout) განსაზღვრისას რესპონსიულობის მისაღწევად სხვადასხვა ტექნიკისა და საზომი ერთეულების გამოყენება შეგვიძლია. ზოგიერთ ელემენტს, მაგალითად ღილაკებს, ყოველთვის არ სჭირდებათ ზომაში რესპონსიულობა, მაგრამ ბევრი სხვა კომპონენტისთვის მომგებიანია პროცენტული მნიშვნელობების ან მოქნილი (flexible) განლაგებების გამოყენება.




React Native-ში რესპონსიული განლაგების შექმნის ერთ-ერთი საუკეთესო გზა Flexbox-ის გამოყენებაა. React Native-ში Flexbox ვებ-ვერსიის მსგავსია, თუმცა არის მცირე განსხვავებები:

    -ყველა ელემენტი ნაგულისხმევად (by default) flex კონტეინერია; ჩვენ არ გვჭირდება display: flex-ის მითითება.

    -ზოგიერთი ნაგულისხმევი მნიშვნელობა შეცვლილია, რათა უკეთ მოერგოს მობილურ აპლიკაციებს.



კულისებში, React Native იყენებს Meta-ს მიერ შექმნილ ღია საწყისი კოდის მქონე განლაგების ძრავს (layout engine), რომელსაც Yoga ჰქვია. Yoga განსაზღვრავს ყუთების (boxes) ზომასა და პოზიციას მოწყობილობის ეკრანის პარამეტრების მიხედვით. ის პირდაპირ არ ხატავს UI ელემენტებს. ამის ნაცვლად, Yoga მუშაობს ჩვენს კომპონენტებთან შესაბამისი განლაგების გამოსათვლელად და შემდეგ ამ განლაგებებს თარგმნის თითოეული პლატფორმის მშობლიურ (native) განლაგების ბრძანებებად.



განლაგებების დინამიურობისთვის ვიყენებთ flex თვისებას. ეს თვისება შვილობილ (child) კომპონენტებს საშუალებას აძლევს, გაფართოვდნენ ან შემცირდნენ, რათა შეავსონ ხელმისაწვდომი სივრცე მთავარი ღერძის (main axis) გასწვრივ. მაგალითად, თუ გვინდა, რომ სამმა ყუთმა სივრცე თანაბრად გაიყოს, შეგვიძლია დავწეროთ:

<View style={styles.layout}>
 <View style={[styles.box, { flex: 1, backgroundColor: 'magenta' }]} />
 <View style={[styles.box, { flex: 1, backgroundColor: 'orange' }]} />
 <View style={[styles.box, { flex: 1, backgroundColor: 'magenta' }]} />
</View>



აქ თითოეული შვილობილი ელემენტი იღებს ხელმისაწვდომი სივრცის თანაბარ წილს. თითოეული ყუთის სიმაღლე გამოითვლება ხელმისაწვდომი სივრცის flex ფაქტორების ჯამზე გაყოფით — ამ შემთხვევაში, სიმაღლე / (1 + 1 + 1) * 1, ასე რომ, თითოეული ყუთი იღებს სივრცის მესამედს.


ჩვენ ასევე შეგვიძლია შევცვალოთ ფაქტორი ერთი ან მეტი შვილობილი ელემენტისთვის, რათა დავარეგულიროთ მათი ზომა სხვებთან მიმართებაში. მაგალითად:

<View style={styles.layout}>
 <View style={[styles.box, { flex: 1, backgroundColor: 'magenta' }]} />
 <View style={[styles.box, { flex: 2, backgroundColor: 'orange' }]} />
 <View style={[styles.box, { flex: 1, backgroundColor: 'magenta' }]} />
</View>


ახლა შუა (ნარინჯისფერ) ყუთს აქვს flex მნიშვნელობა 2, ხოლო დანარჩენებს — 1 (შენიშვნა: ორიგინალ ტექსტში ეწერა ყვითელი ყუთი, თუმცა კოდში მითითებულია ფერი orange). ხელმისაწვდომი სივრცე იყოფა flex ფაქტორების ჯამზე (1 + 2 + 1 = 4). თითოეული flex: 1 შვილობილი ელემენტი იღებს სივრცის 25%-ს, ხოლო flex: 2 შვილობილი ელემენტი იღებს 50%-ს. ეს აადვილებს ისეთი განლაგებების შექმნას, სადაც კომპონენტები დინამიურად ფართოვდება და მცირდება ეკრანის ზომის მიხედვით.



Flexbox-ისა და Yoga განლაგების ძრავის გამოყენებით, ჩვენ შეგვიძლია ავაშენოთ რესპონსიული განლაგებები, რომლებიც შეუფერხებლად ერგება ნებისმიერ მოწყობილობას ან ორიენტაციას, რაც უზრუნველყოფს სტაბილურ და მომხმარებელზე მორგებულ გამოცდილებას ყველგან.





ფლექსბოქსის (Flexbox) გამოყენებას, გარდა იმისა, რომ გვაძლევს სრულ კონტროლს ხელმისაწვდომი სივრცის განაწილებაზე, სხვა უპირატესობებიც აქვს — ჩვენ ასევე შეგვიძლია ვაკონტროლოთ კონტეინერის შიგნით არსებული შვილობილი ელემენტების განლაგების მიმართულება. Expo-სა და React Native-ში ფლექსბოქსის ნაგულისხმევი (default) მიმართულება განსხვავდება ვებ-პლატფორმისგან: აქ ის ვერტიკალურადაა დაყენებული.

არსებობს ორი ძირითადი მიმართულება, რომლითაც შეგვიძლია შვილობილი ელემენტების ასახვა (render) — ჰორიზონტალური (row) ან ვერტიკალური (column). ტერმინოლოგია ცხრილებს უკავშირდება — როდესაც ელემენტებს row მიმართულებით ვათავსებთ, ისინი ჰორიზონტალურად ლაგდებიან. გარდა ამ ორი მიმართულებისა, ჩვენ ასევე შეგვიძლია გამოვიყენოთ შებრუნებული (reversed) მიმართულებებიც:

row: ასახავს შვილობილ ელემენტებს მარცხნიდან მარჯვნივ, ანუ ჰორიზონტალურად.

row-reverse: ასახავს შვილობილ ელემენტებს მარჯვნიდან მარცხნივ, ანუ შებრუნებულ ჰორიზონტალურ მდგომარეობაში.

column: ასახავს შვილობილ ელემენტებს ზემოდან ქვემოთ, ანუ ვერტიკალურად.

column-reverse: ასახავს შვილობილ ელემენტებს ქვემოდან ზემოთ, ანუ შებრუნებულ ვერტიკალურ მდგომარეობაში.
