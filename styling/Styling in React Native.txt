აპლიკაციის დეველოპერებისთვის ერთ-ერთი გავრცელებული გამოწვევა სხვადასხვა მოწყობილობებზე მორგებაა, რადგან თითოეულ მათგანს საკუთარი ეკრანის ზომა, გაბარიტები და ოპერაციული სისტემა გააჩნია. React Native (რომელიც არის ღია საწყისი კოდის მქონე მობილური დეველოპმენტის ფრეიმვორქი და მუშაობს სხვადასხვა ოპერაციულ სისტემაზე, როგორიცაა iOS და Android) ამას აღწევს სტილიზაციის იმ მიდგომით, რომელიც ჩვენთვის კარგად ნაცნობ CSS-ს წააგავს.


React Native-ში style prop-ი (თვისება) კომპონენტების ვიზუალს არეგულირებს. ტრადიციული CSS-ის ნაცვლად, სტილები განისაზღვრება JavaScript ობიექტების გამოყენებით. React Native ამ JavaScript სტილის ობიექტებს აკავშირებს მოწყობილობის გამშვებ გარემოსთან, რითაც უზრუნველყოფს UI-ის (მომხმარებლის ინტერფეისის) რენდერს მოწყობილობაზე.



მოდით, დავიწყოთ საბაზისო style ობიექტის გადაცემით:
<View style={{ width: 100, height: 100, backgroundColor: 'red' }} />




აქ არის თქვენი ტექსტის ქართული თარგმანი (ტექსტიდან ამოღებულია დაკოპირებისას შეცდომით გამოყოლილი ზედმეტი დეტალები, როგორიცაა "Copy to Clipboard" და "Preview: Docs Loading", ტექსტის უკეთ აღსაქმელად):

აპლიკაციის სტილიზაცია მნიშვნელოვან როლს ასრულებს, როგორც ესთეტიკის, ისე ფუნქციონალურობის თვალსაზრისით. სტილიზაციას შეუძლია მიიპყროს მომხმარებლის ყურადღება ყველაზე მნიშვნელოვანი ინფორმაციისკენ, უკარნახოს მას შესაძლო ინტერაქციები და ისე მოაწესრიგოს რთული ინფორმაცია, რომ აპლიკაციის გამოყენება უფრო სასიამოვნო გახადოს.

აპლიკაციის დეველოპერებისთვის ერთ-ერთი გავრცელებული გამოწვევა სხვადასხვა მოწყობილობებზე მორგებაა, რადგან თითოეულ მათგანს საკუთარი ეკრანის ზომა, გაბარიტები და ოპერაციული სისტემა გააჩნია. React Native (რომელიც არის ღია საწყისი კოდის მქონე მობილური დეველოპმენტის ფრეიმვორქი და მუშაობს სხვადასხვა ოპერაციულ სისტემაზე, როგორიცაა iOS და Android) ამას აღწევს სტილიზაციის იმ მიდგომით, რომელიც ჩვენთვის კარგად ნაცნობ CSS-ს წააგავს.

React Native-ში style prop-ი (თვისება) კომპონენტების ვიზუალს არეგულირებს. ტრადიციული CSS-ის ნაცვლად, სტილები განისაზღვრება JavaScript ობიექტების გამოყენებით. React Native ამ JavaScript სტილის ობიექტებს აკავშირებს მოწყობილობის მშობლიურ (native) view თვისებებთან, რითაც უზრუნველყოფს UI-ის (მომხმარებლის ინტერფეისის) რენდერს მოწყობილობაზე.

მოდით, დავიწყოთ საბაზისო style ობიექტის გადაცემით:


<View style={{ width: 100, height: 100, backgroundColor: 'red' }} />

თვისებებში გამოიყენება camelCase: ვწერთ backgroundColor-ს და არა background-color-ს.

საზომი ერთეულების მითითება აუცილებელი არ არის: width, height და სხვა თვისებები იყენებენ პიქსელების სიმკვრივისგან დამოუკიდებელ პიქსელებს (DIPs). (ამას უფრო დეტალურად მომავალ მოდულში განვიხილავთ).
