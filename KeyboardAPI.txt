თუ KeyboardAvoidingView ვიზუალურ განლაგებაზე ზრუნავს, Keyboard API გაძლევს საშუალებას, კოდიდან უბრძანო კლავიატურას, რა გააკეთოს, ან გაიგო, რას აკეთებს ის.


ძირითადი ფუნქციები:

    - Keyboard.dismiss(): ეს ყველაზე ხშირად გამოყენებული ფუნქციაა. ის ხურავს კლავიატურას.
        
        მაგალითი: როცა მომხმარებელი დააჭერს ღილაკს „შენახვა“, ლოგიკურია, რომ კლავიატურა თავისით დაიხუროს.

    -Keyboard.addListener(): ეს გაძლევს საშუალებას „უსმინო“ კლავიატურას. მაგალითად, თუ კლავიატურა ამოიწია, შეგიძლია ეკრანზე რაღაც ელემენტი (მაგალითად, ქვედა მენიუ/Tab Bar) საერთოდ დამალო, რომ ადგილი გამოთავისუფლდეს.



    Keyboard API-ს ყველაზე ხშირი გამოყენება არის კლავიატურის დახურვა (dismiss) და მისი მოსმენა (listeners). მოდი, ორივე შემთხვევა დეტალურად განვიხილოთ პრაქტიკული მაგალითებით.

1. კლავიატურის დახურვა (Keyboard.dismiss)
წარმოიდგინე, გაქვს ფორმა, სადაც მომხმარებელი წერს სახელს. როცა ის დააჭერს ღილაკს "Submit", კლავიატურა ავტომატურად არ იხურება (თუ სპეციალურად არ მიუთითებ). მომხმარებლისთვის მოუხერხებელია, როცა ეკრანის ნახევარი ისევ კლავიატურას უჭირავს, როცა მან საქმე უკვე დაასრულა.

import React from 'react';
import { View, TextInput, Button, Keyboard, StyleSheet } from 'react-native';

const LoginScreen = () => {
  const handleSubmit = () => {
    // აქ ვასრულებთ რაღაც ლოგიკას (მაგ. ავტორიზაციას)
    console.log("მონაცემები გაიგზავნა!");

    // და აი აქ ვიყენებთ API-ს კლავიატურის დასახურად:
    Keyboard.dismiss();
  };

  return (
    <View style={styles.container}>
      <TextInput 
        placeholder="შეიყვანეთ სახელი" 
        style={styles.input} 
      />
      <Button title="გაგზავნა" onPress={handleSubmit} />
    </View>
  );
};
