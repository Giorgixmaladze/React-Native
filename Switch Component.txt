<Switch> კომპონენტი, შეგვიძლია წარმოვიდგინოთ, როგორც <input type="checkbox"> Htmlში. ეს არის უბრალოდ ვიზუალური ჩამრთველ-გამომრთველი",რომელსაც ყოველდღიურად ვხვდებით მობილურის პარამეტრებში


ვებ-დეველოპმენტში თუ checkbox-ის მდგომარეობას checked ატრიბუტით ვმართავთ, React Native-ში <Switch>-ს აქვს value prop-ი. ის იღებს მხოლოდ ლოგიკურ (boolean) მნიშვნელობას — true (ჩართულია) ან false (გამორთულია).



როგორ მუშაობს ეს პრაქტიკაში?
ორი მთავარი ნაწილი, რაც გჭირდება, არის:

    1.value: აკავშირებ შენს React state-თან, რათა კომპონენტმა იცოდეს, ამჟამად ჩართული უნდა იყოს თუ გამორთული.

    2.onValueChange: ეს იგივეა, რაც ჩვეულებრივ React-ში onChange. როცა მომხმარებელი ღილაკს დააკლიკებს (ან თითს გაუსვამს), ეს ფუნქცია ეშვება და გადმოგცემს ახალ მნიშვნელობას (თუ იყო true, გაძლევს false-ს და პირიქით).


მოკლედ რომ შევაჯამოთ: * <TextInput> გამოიყენება, როცა მომხმარებელმა ტექსტი უნდა ჩაწეროს კლავიატურით.


<Switch> გამოიყენება მაშინ, როცა მომხმარებელს უბრალოდ ვეკითხებით "კი თუ არა?" და გვინდა ეს მობილურისთვის ბუნებრივი, ლამაზი გადამრთველით ვაჩვენოთ.



props:

    -value => გადამრთველის მდგომარეობა: true (ჩართული) ან false (გამორთული).

    -onValueChange => ფუნქცია, რომელიც იცვლის მნიშვნელობას გადართვისას. true-ს ხდის false-ად და პირიქით.

    -trackColor => ობიექტი, რომელიც განსაზღვრავს ფონის ფერს. მაგ: {{ false: "red", true: "green" }}

    -thumbColor => თავად იმ მრგვალი ღილაკის ფერი, რომელსაც თითით ვამოძრავებთ.

    -ios_backgroundColor => iOS-ზე, როცა გადამრთველი გამორთულია, მის გარშემო ჩარჩოს ფერს განსაზღვრავს.
