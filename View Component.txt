<View> კომპონენტი:

    React Native-ში <View> ყველაზე ფუნდამენტური კომპონენტია. თუ ვებ-დეველოპმენტიდან მოდიხართ, ის შეგიძლიათ წარმოიდგინოთ როგორც <div>, ხოლო iOS/Android დეველოპმენტში მისი ანალოგებია UIView და android.view.View.

    მოდით, უფრო დეტალურად განვიხილოთ მისი დანიშნულება და მუშაობის პრინციპი:

    1.ძირითადი დანიშნულება:
        - <View> არის კონტეინერი,რომელიც გამოიყენება ეკრანზე ელემენტების დასაჯგუფებლად და განლაგების შესაქმნელად. მას არ აქვს საკუთარი ვიზუალური სახე.


    2.სტილიზაცია და flexbox:
        -React Native-ში განლაგების სამართავად გამოიყენება Flexbox. მნიშვნელოვანია იცოდეთ, რომ ვებისგან განსხვავებით, React Native-ში flexDirection ნაგულისხმევად არის column (ვერტიკალური) და არა row.

        -<View style={{ 
            flex: 1, 
            flexDirection: 'row', // ელემენტები განლაგდება ჰორიზონტალურად
            backgroundColor: 'skyblue',
            justifyContent: 'center', // ჰორიზონტალური გასწორება
            alignItems: 'center'      // ვერტიკალური გასწორება
        }}>
            <View style={{ width: 50, height: 50, backgroundColor: 'red' }} />
            <View style={{ width: 50, height: 50, backgroundColor: 'blue' }} />
        </View>


    3.მნიშვნელოვანი თვისებები(props)

        გარდა სტილისა <View>ს აქვს რამდენიმე მნიშვნელოვანი თვისება:

            -onLayout: ფუნქცია, რომელიც გამოიძახება, როდესაც კომპონენტის ზომა ან პოზიცია გამოითვლება. ეს სასარგებლოა დინამიური გათვლებისთვის.

            -pointerEvents: აკონტროლებს, რეაგირებს თუ არა View შეხებაზე (მაგალითად, none ნიშნავს, რომ შეხება „გაივლის“ მასში და ქვედა ფენაზე იმოქმედებს).

            -nativeID: გამოიყენება ნატივურ კოდთან დასაკავშირებლად.

            -Accessibility Props: თვისებები შეზღუდული შესაძლებლობების მქონე პირებისთვის (მაგ. accessible={true}, accessibilityLabel="მთავარი კონტეინერი").

    4.რისი გაკეთება არ შეიძლება <View>-ში?

        -პირდაპირი ტექსტი: არასდროს ჩაწეროთ ტექსტი პირდაპირ <View>-ში. ეს გამოიწვევს შეცდომას. ტექსტისთვის ყოველთვის გამოიყენეთ <Text>.

        -სქროლვა: <View> არ სქროლავს. თუ კონტენტი ეკრანზე არ ეტევა, უნდა გამოიყენოთ <ScrollView>.
